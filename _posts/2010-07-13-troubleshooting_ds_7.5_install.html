1. Error Msg:

 
##I TFCP 000008 15:53:21(004) <transform> Error when checking composite operator: cxx  -O   -IC:/Ascential/DataStage/PXEngine/include -W/TP -W/EHa -DAPT_USE_ANSI_IOSTREAMS -c sourcefilename.C -o outputname.o
 
Solution: There's no C++ compiler installed on the Windows server. The Transformer Stage in DataStage 7.5 needs Visual Studio 2003 C++ compiler. If you don't have Visual Studio 2003, Visual C++ Toolkit 2003 will also work. It is a free version of the same C++ compiler shipped with Visual Studio .NET 2003 without the IDE, though it is no longer available and now superseded by the Express Editions. After installing the compiler, remember to config the Windows system environment (Path, Include, Lib). The environment value should be modified before DataStage Server is installed.
 
2. Error Msg:
 
##W TBLD 000000 15:53:21(003) <main_program> Error when checking composite operator: Output from subprocess: cxx: file 'C:/PROGRA~1/MKSTOO~1/etc/cxx.ccg': The system cannot find the file specified.
 
Solution: Copy cxx.ccg file from C:\Program Files\MKS Toolkit\etc\nutccg\ to C:\Program Files\MKS Toolkit\etc\
 
3. Error Msg:
 
##I TBLD 000000 10:52:58(007) <main_program> Error when checking composite operator: Output from subprocess: LINK : fatal error LNK1181: cannot open input file 'User32.lib'
 
Solution: Windows Platform SDK should be installed. The Windows system environment should also be updated.
